# Changelog

Все изменения проекта Liquid Torrent документируются в этом файле.

## [1.3.2] — 2026-05-17

### 🐛 Исправления (по результатам аудита qBittorrent)

- **Pause All / Resume All** — кнопки теперь работают корректно (переключают каждый торрент по отдельности вместо session.pause(), который не обновлял UI-статус)
- **Торренты не терялись при перезапуске** — данные всегда сохраняются в `%APPDATA%\liquid-torrent` (portable exe извлекался во временный каталог, что приводило к потере данных)
- **Ограничение скорости** — libtorrent ожидает `0` для безлимита (было `-1`, что могло вести к некорректному поведению). Выравнено с qBittorrent `cleanLimitValue()`
- **Глобальные лимиты** — настройки `download_rate_limit` и `upload_rate_limit` теперь используют `0` для безлимита
- **saveTorrentsState()** — исправлен критический баг: вместо сохранения вызывался shutdown сидкара, что убивало движок

### ⚡ Улучшения

- Добавлена команда `save` в sidecar — безопасное сохранение состояния без остановки движка
- Код проверен по исходному коду qBittorrent (`cleanLimitValue`, `pause_all/resume_all`, `saveResumeData`)

## [1.3.1] — 2026-05-17

### 🔧 Стабильность

- **Авто-сохранение каждые 30 сек** — торренты не теряются даже при аварийном закрытии
- **Graceful shutdown** — при закрытии приложение ждёт до 13 секунд, пока Python sidecar сохранит resume data и torrents.json
- **Лимит отдачи убран** — `maxUploadSpeed: -1` (было 5 МБ/с), libtorrent имеет собственный congestion control
- **Resume data** каждые 5 минут — мгновенное восстановление торрентов без перехеширования

### 📦 Сборка

- Портативный EXE: `LiquidTorrent-1.3.1-portable.exe`
- Обновлён README с актуальной архитектурой (libtorrent sidecar)

## [1.3.0] — 2026-05-17

### 🏗️ Архитектура

- **Переход на libtorrent 2.0.11 (C++)** — WebTorrent (JS) заменён на Python sidecar с нативным libtorrent. Скорость выросла с 5-15 МБ/с до 15-35+ МБ/с
- **Python sidecar** работает в отдельном процессе через `child_process.spawn` + stdin/stdout JSON-RPC — UI Electron **никогда не зависает**
- **PyInstaller bundle** — sidecar упакован в `torrent_sidecar.exe` (~10.5 МБ), Python на целевом ПК не нужен

### 🚀 Оптимизация производительности

- **Кэш метаданных** — файлы/трекеры кэшируются один раз, не пересоздаются каждый тик
- **Лёгкий polling** — UI запрашивает только лёгкие данные (без файлов/трекеров)
- **React.memo на TorrentCard** — карточки не перерисовываются если данные не изменились
- **Мемоизация Sidebar** — счётчики фильтров обёрнуты в `useMemo`
- **Параллельное восстановление** — торренты при запуске загружаются одновременно

### 🐛 Исправления

- Исправлена захардкоженная версия в AboutModal (теперь `__APP_VERSION__`)
- Удалены неиспользуемые импорты (`nativeTheme`, `Download`, `Upload`, `CheckSquare`)

## [1.2.0] — 2026-05-16

### ✨ Первый релиз

- Полноценный торрент-клиент на Electron + React + WebTorrent
- Тёмная тема с неоновыми акцентами и glassmorphism
- Поддержка .torrent файлов и магнет-ссылок
- Системный трей с контекстным меню
- Дерево файлов торрента
- График скорости в реальном времени
- Индивидуальные лимиты скорости для торрентов
- Ассоциация с файлами `.torrent` и протоколом `magnet:`
- Настройки: лимиты скорости, авто-стоп раздачи, автозапуск
- Портативный EXE без установки
